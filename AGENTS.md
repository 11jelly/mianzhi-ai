# 开发约束

1. 后续每完成一个阶段，必须生成 `docs/progress/phase-x.md`。
2. 阶段文档必须包含：
   - 完成功能；
   - 修改文件；
   - 启动命令；
   - 测试命令；
   - 测试结果；
   - 已知限制；
   - 下一步建议；
   - 建议 Git commit message。
3. 不执行 `git add`、`git commit`、`git push`。
4. 不把密码、Token、API Key 写入源码、README、日志或 Git。
5. 每次开发先阅读 README、AGENTS.md、当前 `docs/progress` 最新记录。
6. 不跨阶段实现未要求的业务功能。
